最終更新: 2026-04-21
対象: MVP (v1)

## 画面一覧

| #   | 画面          | ルート           | 用途                                 |
| --- | ------------- | ---------------- | ------------------------------------ |
| 1   | ホーム / 一覧 | `/`              | 最近のレシート一覧、月合計、追加導線 |
| 2   | アップロード  | `/upload`        | 撮影 or ファイル選択                 |
| 3   | 抽出結果確認  | `/upload/review` | Gemini 抽出結果の確認・編集・保存    |

## 2. アップロード `/upload`

```
┌─────────────────────────────────┐
│ ← 戻る    レシートを追加        │
├─────────────────────────────────┤
│                                 │
│   ┌─────────────────────┐      │
│   │                     │      │
│   │     📷              │      │
│   │   カメラで撮影       │      │  ← 主ボタン（大）
│   │                     │      │
│   └─────────────────────┘      │
│                                 │
│   ┌─────────────────────┐      │
│   │  🖼 画像を選択       │      │  ← 副ボタン
│   └─────────────────────┘      │
│                                 │
│   選択中: receipt.jpg (1.2MB)   │  ← 選択後に表示
│                                 │
│   [ 読み取る ]                   │  ← 選択後に活性化
│                                 │
└─────────────────────────────────┘
         │
         │ 読み取り中は全画面ローディング
         ▼
┌─────────────────────────────────┐
│    📄 解析中...                  │
│    ──────────────── 70%         │
│    画像をアップロード ✓          │
│    Gemini に送信中 ...           │
└─────────────────────────────────┘
```

フロー:

1. 撮影 or 選択 → クライアントで HEIC→JPEG・長辺 2048px 縮小
2. `PUT /api/upload` で R2 に保存（WebP 圧縮は Worker 側）
3. `POST /api/extract` で Gemini 呼び出し
4. 成功 → `/upload/review` に遷移

エラーハンドリング:

- ネットワーク失敗 → その場でリトライボタン
- Gemini 抽出失敗 → 空のレビュー画面に画像だけ出し、手入力にフォールバック

## 3. 抽出結果確認・編集 `/upload/review`

スマホは縦積み、PC は左右 2 カラム。

```
┌─────────────────────────────────┐
│ ← 戻る    内容を確認             │
├─────────────────────────────────┤
│ ┌──────────────┐                │
│ │              │                │
│ │   [画像]     │   タップで拡大  │
│ │              │                │
│ └──────────────┘                │
├─────────────────────────────────┤
│ 店舗                             │
│ ┌──────────────────────────┐   │
│ │ セブンイレブン 渋谷店      │   │
│ └──────────────────────────┘   │
│                                 │
│ 日時                             │
│ ┌──────────────────────────┐   │
│ │ 2026-04-20 18:22          │   │
│ └──────────────────────────┘   │
│                                 │
│ 合計                             │
│ ┌──────────────────────────┐   │
│ │ ¥ 842                     │   │
│ └──────────────────────────┘   │
│                                 │
│ カテゴリ                         │
│ [食費 ▾]  ← セレクト            │
│                                 │
│ ⚠ 似たレシートが既にあります    │  ← 重複候補ヒット時のみ
│   04/20 セブンイレブン渋谷 ¥842 │
│   [ 上書きしない ] [ 確認する ]  │
├─────────────────────────────────┤
│ [ 破棄 ]          [ 保存 ]      │
└─────────────────────────────────┘
```

挙動:

- `confidence < 0.7` のフィールドは黄色枠で強調
- 保存 → `status = confirmed` で SurrealDB に書き込み → `/` に戻る
- 破棄 → R2 の画像も削除（`status = rejected` のまま残す選択肢もあり）

## 共通コンポーネント

- **ヘッダ**: タイトル + ハンバーガーメニュー。戻るボタンは画面ごとに左上
- **フッタ**: MVP では不要。v2 でタブバー（一覧 / 追加 / 集計）検討
- **トースト**: 保存成功・削除成功・エラーは画面下部に 3 秒トースト
- **ダイアログ**: 削除確認のみ。それ以外はインライン表示

## 状態遷移

```
  [/]
   │  ＋
   ▼
[/upload] ──失敗──> エラー表示 → リトライ
   │  読み取り成功
   ▼
[/upload/review]
   │  保存              破棄
   ├────────────────────┐
   ▼                    ▼
  [/]                  [/]
```

## v2 で追加予定の差分

- レビュー画面に `items[]` テーブル行を追加（追加・削除・金額自動合計）
- 一覧に全文検索ボックス
- 設定画面にカテゴリ辞書（`store_category_hint`）の閲覧・編集
- 再OCRボタン（詳細画面の `⋮` メニュー内）
